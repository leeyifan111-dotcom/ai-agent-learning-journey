# 📅 复习排期

> 学习只前进不回头 = 三十天后忘 90%。本文件维护**主动召回（active recall）**节奏，到期项必须按时做。

---

## 🔁 三层复习机制

| 层级              | 触发时机          | 形式                                                                                                         | 时长     |
| ----------------- | ----------------- | ------------------------------------------------------------------------------------------------------------ | -------- |
| **L1 · 次日快查** | 每天开工前        | 扫**昨天**的 daily README，能否用一句话复述每篇核心收获？不能→重读 read_feelings 30 秒                       | 5 分钟   |
| **L2 · 周复习**   | 一周后            | **默写** read_feelings——盖住原文，按框架自己重写，再对照差异                                                 | 30 分钟  |
| **L3 · 阶段总复** | 进入下一 Phase 前 | 把整个 Phase 的 read_feelings 串成**心智地图长文**，必须能从某个钩子（如"Day 1 反复横跳"）出发串到所有知识点 | 1-2 小时 |
| **L4 · 月回访**   | 30 天后           | 重做 L1（用一句话复述能否答出来），不行→打回 L2                                                              | 5 分钟   |

**关键原则：**

- ❌ 不是"重读笔记"——眼睛过一遍效果接近零
- ✅ 是**主动召回**——盖住原文先自己说/写，再对照差异
- ✅ **做中复**——把所学回 tiny-agent-loop 里用代码再实现一次（最强复习，如 v0.4 改造就是 #059/#061/#062 的复习）

## 📝 默写模板（L2 用）

复习 daily/YYYY-MM-DD 时，新建 `daily/YYYY-MM-DD/review.md`，按这个模板写：

```markdown
# YYYY-MM-DD 复习（L2 周复习）

## 凭记忆默写（盖住 read_feelings）

[在这里凭记忆把每篇的核心写出来]

## 对照差异

| 篇   | 我默写的 | 原文有但我忘了 | 我新加的（说明已内化） |
| ---- | -------- | -------------- | ---------------------- |
| #xxx |          |                |                        |

## 反思

- 哪些点彻底忘了？→ 追加到 review-log 的「待复习」
- 哪些点反而比上次理解更深？→ 加进 read_feelings 增补
```
